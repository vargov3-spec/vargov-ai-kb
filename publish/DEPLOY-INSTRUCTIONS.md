# Публикация на vargov.ru (Tilda) — пошагово

Все файлы для загрузки лежат в этой папке (`publish/`). Порядок:

## 1. Загрузить llms.txt в корень домена
1. Зайдите в Tilda → выберите проект vargov.ru → **Настройки сайта** (шестерёнка) → **Ещё** (More).
2. Найдите раздел про кастомные файлы в корне сайта (в разных версиях Tilda называется «Свои файлы» / «robots.txt и другие файлы» / «Custom files»). Обычно там же, где можно переопределить `robots.txt` или загрузить `ads.txt`.
3. Загрузите файл `llms.txt` из этой папки — он должен стать доступен по адресу `https://vargov.ru/llms.txt`.
4. Если такого раздела в вашем тарифе/интерфейсе нет — напишите мне, дам альтернативный вариант (через отдельную статическую страницу Tilda с этим текстом, или через редирект).

## 2. Загрузить датасеты
Так же, через «Свои файлы», загрузите:
- `vargov-products.json` → должен быть доступен на `https://vargov.ru/vargov-products.json`
- `vargov-schema-by-path.json` → должен быть доступен на `https://vargov.ru/vargov-schema-by-path.json`

Проверка после загрузки: откройте оба адреса в браузере — должен открываться сырой JSON, а не 404.

## 3. Вставить код структурированных данных (Schema.org/JSON-LD)
1. Откройте содержимое файла `tilda-head-snippet.html` из этой папки.
2. В Tilda: **Настройки сайта → Ещё → код в `<head>` для всех страниц сайта** (в интерфейсе Tilda это поле обычно подписано как код аналитики/счётчиков — годится то же поле, Tilda вставляет его в `<head>` каждой страницы).
3. Вставьте содержимое файла целиком и сохраните.
4. Этот код делает две вещи:
   - Добавляет на каждую страницу сайта общую разметку **Organization/Brand** (статично).
   - Подгружает `vargov-schema-by-path.json` и, если текущий URL страницы совпадает с одним из 645 товаров, добавляет для неё персональную разметку **Product** (артикул, изображения, ссылка, бренд).

## 4. Проверка после публикации
- Откройте любую карточку товара на сайте, посмотрите исходный код страницы (Ctrl+U) → должно быть два `<script type="application/ld+json">` — один Organization, второй Product с `sku` этого товара.
- Прогоните несколько URL через Google Rich Results Test (https://search.google.com/test/rich-results) — вставляете туда живой URL страницы.
- Проверьте `https://vargov.ru/llms.txt` — обычный текст открывается напрямую в браузере.

## 5. Что делать при обновлении каталога
Если на сайте появляются новые товары или меняются старые — датасеты (`vargov-products.json`, `vargov-schema-by-path.json`) нужно пересобрать заново (сканированием сайта) и перезалить в Tilda. Это можно оформить как отдельную повторяемую задачу — скажите, и настрою.
